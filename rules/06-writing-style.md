# Writing Style

Always-loaded authoring rules for text produced or edited by any agent. Applies to ALL agents, every session.

## No Em-Dashes (forward-looking, all documents)

- The em-dash (the long dash, Unicode U+2014) and the en-dash (U+2013) are forbidden in any newly authored or edited document text: prose, plans, reports, directives, memory bodies, commit messages, LaTeX, markdown, code comments. Use semicolons, colons, commas, periods, or parentheses instead.
- Plain hyphens (the ordinary "-", U+002D) are fine in compound words (read-only, toto-migration) and in markdown structure (horizontal rules, table separators, list bullets).
- Forward-looking: existing documents are grandfathered. Do NOT mass-prune pre-existing em-dashes. When editing a grandfathered doc, introduce no NEW em-dashes; purge pre-existing ones only during a full rewrite of that doc.
- Applies to ALL agents (meta, scaf, orch, workers). Ratified by Abe 2026-07-01. Supersedes the academic-only scope of the instance/meta memory "European academic writing style".
